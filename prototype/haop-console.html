Companion Console Review Before Accept

Companion console for governed background delegation.

HAOP Console is a management and oversight layer for delegated background agents. It tracks permission boundaries, generates reusable delegation prompts, preserves audit-friendly log chains, and makes risk posture explicit before any accepted change reaches the main workspace.

Daily work stays in the foreground AI coding harness. HAOP Console does not replace Windsurf, Cascade, Cursor, Claude Code, VS Code Agent, or any front-stage coding environment.

Permission Profiles

Risk Warnings

Default-visible reminders that reinforce safe delegation and bounded execution.

`danger-full-access` disabled by default

Unbounded execution should remain off unless an explicit approval path exists and the review chain captures the reason.

Write scope required

Any write-capable run must declare its allowed paths before a prompt is copied or fallback command is generated.

Secrets blocked

Credential files, tokens, `auth.json`, and private keys stay outside delegated read scope and should never be surfaced here.

Review required

Accepted changes demand human review of prompt, diff, validation, and final answer artifacts even after a successful run.

Default deny

Scope before action

Write-capable prompts and fallback commands should always declare allowed files or directories before generation.

Secrets never delegated

Credential material is out of scope for this console and should not appear in prompts, logs, or review packages.

Human acceptance gate
